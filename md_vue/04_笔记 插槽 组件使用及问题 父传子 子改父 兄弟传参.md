## 组件


### 组件使用


1.  调用`Vue.extend()`方法创建组件构造器  
创建组件构造器时，传入template代表我们自定义组件的模板，以后直接使用`Vue.component(...,{template:...})` 语法糖即可 
2.  调用`Vue.component()`方法注册组件 
3.  在Vue实例的作用范围内使用组件 



**组件特点**



+ 每个组件就是一个自定义标签
+ 可复用
+ 方便维护
+ 方便拆分
+ 每个组件作用域隔离（互不干扰）
+ 有完整的声明周期
+ 有自己的响应式数据和各种方法（事件）



**组件模板抽离方法**



```html
<!-- script标签 -->
<script type="text/x-template" id="first">
	<span>1111</span>
</script>

<!-- template标签 -->
<template id="first">
	<span>1111</span>
</template>
```



### 全局组件


> HTML 元素的属性名是大小写不敏感的，但是 JavaScript 对象的属性名是大小写敏感的。转换规则是，转为 JavaScript 属性名时，一律采用小写。如果属性名包括多个单词，则采用骆驼拼写法，即从第二个单词开始，每个单词的首字母采用大写
>



全局组件：无须单独引用或配置，直接在大组件中调取全局组件即可



	`Vue.component(componentName, options)`



+ options可以使用的有Vue实例的大部分（data、methods、生命周期函数...）
+ 每调用一次组件都是创建一个单独的Vue实例



```html
<div id="app">
    <my-button></my-button>
</div>

<script>
    // PasalCase： 单词首字母大写，也是基于kebab调取
    // 调取的时候，会把所有组件的单词渲染为小写字母（命名时除PasalCase模式外，都要把组件名设置为小写，调取组件的时候可以是大写也可以是小写，最后都是按照小写渲染）
    Vue.component("MyButton", {
        // 必须设置template或render函数：来规定当前组件的渲染视图
        template: `<button>按钮</button>`
    })
	// kebab-case： 横线作为分隔符，只能基于kebab调取
    Vue.component("my-button", {
        template: `<button>按钮</button>`
    })

    // 如果在template模板中可以使用Pasal方式调取
    Vue.component("ss", {
        template: `<p>组件<MyButton/></p>`
    })
</script>
```



**组件的调用：**



1. 双闭合标签
2. 单闭合标签，这种形式其实是不规范的，当前组件可以正常渲染，但下边的东西就不能正常渲染了



**组件的名字的注意事项：**



1. 给组件起名字时，如果名字开头字母大写，使用的时候可以不区分大小写
2. 组件名字不能全是大写（COM不允许）
3. 起名字时如果是驼峰命名法，那使用的时候以`-`隔开使用 （indexItem->index-item）
4. 组件名字不能和h5标签名重复

## slot插槽


+ slot插槽的使用可以让组件变的个性化，在使用组件的时候，在组件标签内传递一些东西到组件内部（在组件内部使用slot标签进行接收）
+ slot插槽可以使用一个，也可以使用多个，如果使用多个的话就是，使用的时候在组件里写slot时，要使用name给插槽起一个名字
+ 在给插槽插入东西的时候，要拿template包起来，并且在template行间写一个`v-slot:...` `<template v-slot:...></template>`



```html
<div id="app">
    <my-button>
        <template v-slot:before><em>第一段内容</em></template>
        <!-- v-slot缩写：# -->
        <template #after><strong>第二段内容</strong></template>
    </my-button>
</div>
<template id="myButtonTemplate">
    <div>
        <!-- slot插槽：再调取组件的时候，写在组件中间部分的内容获取到，防止到组件的某个位置，依托这种形式，可以让当前组件具备更强的扩展性 -->
        <slot name="before"></slot>
        <button>消息</button>
        <slot name="after"></slot>
    </div>
</template>
<script>
    // 组件实例
    Vue.component("MyButton", {
        template: "#myButtonTemplate"
    });
    // 根实例
    let vm = new Vue({
        el: "#app",
    })
</script>
```



1. 插槽的基本使用 `<slot></slot>`
2. 插槽的默认值 `<slot>button<slot>`
3. 如果有多个值， 同时放入懂啊组件进行替换时，一起作为替换元素

### 编译作用域


+ 官方准则：**父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的**



```html
<div id="app">
  <cpn v-show="isShow"></cpn>
</div>
<template id="cpn">
  <div>
    <h2>子组件</h2>
    <p>内容区域</p>
  </div>
</template>

<script>
  Vue.component("cpn", {
    template: "#cpn",
    data() {
      return {
        isShow: false,
      };
    },
  });
  let vm = new Vue({
    el: "#app",
    data: {
      isShow: true,
    },
  });
</script>
```

## 组件中data必须是函数


组件是一个单独的模块封装，组件数据保存在哪里呢？



+ Vue组件有自己保存数据的地方（组件不可以访问 Vue实例数据）



```html
<div id="app">
    <my-com></my-com>
</div>
<template id="myCom">
    <div>消息{{msg}}</div>
</template>

<script>
    // [Vue warn]: Property or method "msg" is not defined on the instance but referenced during render.
    let vm = new Vue({
        el: "#app",
        data: {
            msg: "hello world"
        },
        components: {
            "my-com": {
                template: "#myCom"
            }
        }
    })
</script>
```



组件对象也有data属性、methods属性（很像Vue实例），因为组件原型指向Vue实例



+ **注意：组件里的data必须是一个函数**，因为函数每次返回的都是一个新对象，所以相同组件之间不会影响（相当于形成一个闭包，让返回的对象是当前实例的私有的）



```javascript
// 如果使用 data:{} 会报错
The "data" option should be a function that returns a per-instance value in component definitions.
```



## 组件之间相互通信


### 父传子


在开发中，往往一些数据需要从上层传递到下层



+ 比如一个页面中，我们从服务器请求到了很多数据
+ 其中一部分数据，并非是我们整个页面的大组件展示的，而是需要下面的子组件展示
+ 这时候，并不会让子组件再发送一个网络请求，而是直接让**大组件（父组件）**将数据传递给**小组件（子组件）**



Vue中如何进行父子组件的通信？



+ **通过props（properties）向子组件传递数据**
+ 通过事件向父组件发送消息（$emit Events）



### props


+ 子组件设置props来接收父组件基于属性传递进来的信息：在props中注册的属性和data一样，也会挂载到当前实例上（是响应式数据，可控制视图渲染）`this.title/{{title}}`



```javascript
props: ["title"]
```



**props的值有两种方式**



+ 字符串数组，数组中的字符串就是传递时的名称
+ 对象，对象可以设置传递时的类型、默认值等



### props命名问题


1. 父组件传递的属性名是kebab-case格式，子组件在注册的时候应按照**camelCase驼峰命名**/**PasalCase单词首字母大写**来接收和使用（尽可能不传kebab-case模式）
2. 属性名传递的是大写的，其实也是按照小写的来传递，所以props中注册和使用的时候都按照小写来处理即可（尽可能都传小写字母）



```html
<div id="app">
    <!-- :AAA -> typeof this.aaa -> number -->
    <vote title="hello" data-index="0" AAA="1"></vote>
</div>

<template id="vote">
    <div class="container">
        <h2>{{title}}</h2>
        <!-- PasalCase模式 单词首字母大写 -->
        {{DataIndex}}
        <!-- camelCase模式 驼峰命名 -->
        <!-- {{dataIndex}} -->
        {{aaa}}
    </div>
</template>

<script>
    let vote = {
        template: '#vote',
        props: ['title', "DataIndex", "aaa"],
        // props: ['title', "dataIndex", "aaa"],
        created() {
            console.log(typeof this.aaa); // string
        }
    };

    let vm = new Vue({
        el: '#app',
        components: {
            vote
        }
    });
</script>
```



+ 可以把子组件当做一个标签，可以设置id/class/style等内置属性，这些属性也会传递给子组件，vue帮我们处理好，该合并的合并，该覆盖的覆盖，无需再proos中注册处理



### props数据验证


父组件把信息传递给子组件：



+ **默认传递给子组件的属性值都是字符串格式**，如果需要传递数据的格式是数据本身应该具备的格式，需要基于v-bind实现传递



> 如果数据类型不匹配，仅仅会在控制台抛出警告，但是不会阻止代码运行
>



1.  类型限制 

```javascript
props: {
    ss: String,
}
```

 

2.  提供默认值 

```javascript
props: {
	ss: {
		type: [String, Boolean, Number], 
		default: 'my-default',
	}
}
```

 

3.  必填项 

```javascript
props: {
	ss: {
		type: String,
    required: true,
	}
}
```

 

4.  注意：类型是对象或数组时，默认值必须是一个函数 

```javascript
props: {
	ss: {
		type: Array,
		default() {
            return []
		}
	}
}
```

 

5.  自定义验证规则 

```javascript
props: {
	ss: {
		type: [String, Number], 
		// val是传递的属性值，在根树枝，根据自己的规则，返回true和false来验证传递的值是否符合规则
		validator(val) {
    	return val >= 0;
		}
	}
}
```

 



### 子改父：this.$emit


+  订阅自定义事件：调用组件的时候基于属性传递一个方法（父） 
+  `this.$emit()`  
第一个参数：自定义事件类型  
后面的参数：通知方法执行的时候，给方法传递的参数 



```html
<div id="app">
    <p>总票数：<span v-text="num"></span></p>
    <!-- 基于自定义事件，把父组件的某个方法注册到任务队列中 -->
    <btn @changeparent="changeparent"></btn>
</div>
<template id="btnTemplate">
    <div>
        <button @click="handle('support')">支持</button>
        <button @click="handle('oppose')">反对</button>
    </div>
</template>

<script>
    let btn = {
        template: "#btnTemplate",
        methods: {
            handle(type) {
                // 通知任务队列中@changeparent这个自定义事件执行
                this.$emit("changeparent", type);
            }
        }
    }
    let vm = new Vue({
        el: "#app",
        data: {
            num: 0
        },
        methods: {
            changeparent(type) {
                // this.$emit通知执行的时候传递的参数信息
                this.num++;
            }
        },
        components: {
            btn
        }
    })
</script>
```



### 兄弟组件传参


`@xxx === eventBus.$on("xxx")`



```html
<div id="app">
    <vote></vote>
    <vote></vote>
</div>
<template id="vote">
    <div>
        <p>总票数：<span v-text="num"></span></p>
        <rat :eventbus="eventbus"></rat>
        <!-- 基于自定义事件，把父组件的某个方法注册到任务队列中 -->
        <btn @changeparent="changeparent" :eventbus="eventbus"></btn>
    </div>
</template>
<template id="btnTemplate">
    <div>
        <button @click="handle('support')">支持</button>
        <button @click="handle('oppose')">反对</button>
    </div>
</template>

<template id="ratTemplate">
    <div>
        <p>支持人数：<span v-text="supportNum"></span></p>
        <p>反对人数：<span v-text="opposeNum"></span></p>
    </div>
</template>

<script>
    let btn = {
        template: "#btnTemplate",
        props: ["eventbus"],
        methods: {
            handle(type) {
                // 通知任务队列中@changeparent这个自定义事件执行：第一个参数是自定义事件类型，后面的参数都是通知方法执行的时候，给方法传递的参数
                this.$emit("changeparent", type);
                // 通知eventbus任务队列中changesupport自定义事件执行
                this.eventbus.$emit("changesupport", type);
            }
        }
    }
    let rat = {
        template: "#ratTemplate",
        props: ["eventbus"],
        data() {
            return {
                supportNum: 0,
                opposeNum: 0
            }
        },
        methods: {
            changeNum(type) {
                type === "support" ? this.supportNum++ : this.opposeNum++;
            }
        },
        created() {
            // 基于$on创建一个自定义事件：把指定的方法放到任务队列中
            this.eventbus.$on("changesupport", this.changeNum)
        }
    }

    let vote = {
        template: "#vote",
        data() {
            return {
                num: 0,
                eventbus: new Vue()
            }
        },
        methods: {
            changeparent() {
                this.num++;
            }
        },
        components: {
            btn,
            rat
        }
    }
    let vm = new Vue({
        el: "#app",
        components: {
            vote
        }
    })
</script>
```



## 单向数据流


所有prop都使得其父子prop之间形成一个单向下行绑定：父级prop的更新会向下流动到子组件中，但反过来不行



Vue的父组件和子组件生命周期钩子函数执行顺序如下：



+ 加载渲染过程：父 `beforeCreate` ->父 `created` ->父 `beforeMount` ->子`beforeCreate` ->子 `created` ->子 `beforeMount` ->子 `mounted` -> 父 `mounted`
+ 子组件更新过程：父 `beforeUpdate` ->子 `beforeUpdate`->子 `updated` ->父 `updated`
+ 父组件更新过程：父 `beforeUpdate` ->父 `updated`
+ 销毁过程：父 `beforeDestroy` -> 子 `beforeDestroy` -> 子 `destroved` -> 父 `destroyed`



```javascript
Vue.component("father", {
    template: `<div>father组件<son></son></div>`,
    beforeCreate() {
        console.log("beforeCreate--father");
    },
    created() {
        console.log("created--father");
    },
    beforeMount() {
        console.log("beforeMount--father");
    },
    mounted() {
        console.log("mounted--father");
    }
})
Vue.component("son", {
    template: `<div>son组件</div>`,
    beforeCreate() {
        console.log("beforeCreate--son");
    },
    created() {
        console.log("created--son");
    },
    beforeMount() {
        console.log("beforeMount--son");
    },
    mounted() {
        console.log("mounted--son");
    }
})
// beforeCreate--father
// created--father
// beforeMount--father
// beforeCreate--son
// created--son
// beforeMount--son
// mounted--son
// mounted--father
```



每次父级组件发生更新时，子组件中所有的prop都将会刷新为最新的值。这意味着你不应该在一个子组件内部改变prop。如果你这样做了，Vue会在浏览器的控制台发出警告。子组件想修改时，只能通过$emit派发一个自定义事件，父组件接收到后，由父组件修改。有两种常见的视图改变一个prop的情形



+ 这个prop用来传递一个初始值；这个子组件接下来希望将其作为一个本地的prop数据来使用。在这种情况下，最好定义一个本地的data属性并将这个prop用作其初始值
+ 这个prop以一种原始的值传入且需要进行转换。这种情况下，最好使用这个prop来定义一个计算属性
